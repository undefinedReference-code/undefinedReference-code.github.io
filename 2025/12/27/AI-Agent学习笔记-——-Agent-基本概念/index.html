AI Agent学习笔记 —— Agent 基本概念

AI Agent学习笔记 —— Agent 基本概念

翻转互动模式：当AI从回答者变身为提问者，一切才刚刚开始

在人与AI的传统互动中，我们早已习惯了一个固定模式：​​人类提问，AI回答。无论是“明天的天气如何？”还是“请写一首关于春天的诗”，我们都是那个发出指令、开启对话的主导者。AI则扮演着一位知识渊博、有问必答的“助手”，在收到清晰的指令后，尽其所能地给出最佳回应。

然而，一个强大而巧妙的应用技巧，正在悄然改变这一格局——​Flipped Interaction Pattern（翻转互动模式） 。它通过一个简单的角色对调，将AI从被动的回答者，转变为主动的提问者和引导者。这一翻转，不仅是对话流程的变化，更是AI从“工具”迈向“智能体（Agent）”的关键起点。

它的核心在于：当人类的需求本身是模糊或复杂时，最有效的帮助并非直接给出答案，而是先帮助人类厘清问题本身。 大语言模型所具备的理解上下文、灵活推理的能力，使其能够胜任这个“引导者”的角色。它可以根据我们给出的初步意图，主动规划出获取关键信息的路径，通过一系列有逻辑的提问，在收集到足够的信息之后，再做出决策。

让我们看一个具体的例子。假设你想在纳什维尔找一家今晚用餐的餐厅，但自己还没想清楚所有细节。传统模式下，你可能需要费力地罗列出所有要求；而在翻转模式下，你只需将初步意图交给AI：

我：请逐一问我问题，以便收集足够的信息，为我在田纳西州纳什维尔推荐一家今晚用餐的餐厅。请提出第一个问题。
AI：你今晚想吃什么菜系？
我：德州-墨西哥菜。
AI：你有什么饮食限制或偏好吗？
我：没有。
AI：你更喜欢休闲餐饮还是更高级的体验？
我：呃…适合带孩子的。
AI：你喜欢有户外座位还是室内座位的餐厅，或者无所谓？
我：无所谓，反正下雨！

从上面的例子可以看出，对话的主导权发生了转移。AI不再是等待一个完美指令的工具，而是成为了协作思考的伙伴。它主动构建了一个动态的信息收集框架，每一个新问题都基于之前的回答，整个过程自然、高效，最终汇聚成一份个性化的完美推荐。

智能体循环（The Agent Loop）—— 从思考到行动的自动化引擎

在上一章节我们探讨了“翻转互动模式”，看到了AI如何从被动的回答者转变为主动的提问者，以引导对话、澄清需求。这揭示了智能体（Agent）的一个基本特质：​为达成目标而进行动态规划与交互。

然而，在餐厅推荐的例子中，AI的“行动”仅限于提出下一个问题，其“执行”环节（例如实际查询数据库、筛选餐厅）仍需要人类用户自己在脑海中完成。如果我们希望AI能真正自主地完成任务，比如自动填写报销单、自动生成并执行数据查询，就需要一套更强大的机制。这正是智能体循环（Agent Loop） 所要解决的问题。

在一些任务中，我们希望Agent可以自动通过我们的API去“询问”并操作周围的环境（如数据库、文件系统、网络服务），从环境中获取反馈并处理结果，中间完全不需要人类的参与。下图（图1）就描绘了这一理想场景：人类发出“帮我报销这笔交通费”的指令后，AI智能体自动与计算机（代表一个表格应用）进行多轮交互，最终完成任务。

此图展示了智能体在收到“Help me add this travel expense”指令后，自动执行“列出表格行”和“添加新行”两个动作，与计算机系统协作完成任务的过程。这体现了智能体超越对话，能直接驱动外部工具的能力

循环的齿轮：思考、行动、观察

如何实现图1中的自动化？其核心是一个由大语言模型驱动的循环过程。这个循环包含了几个核心阶段，它们共同构成了智能体的“思考-行动”闭环。

  • 思考：模型推理与任务分解

在此阶段，大语言模型扮演着“大脑”的角色。它会分析用户的整体目标（例如“帮我报销这笔交通费”），并将其​分解为一系列逻辑子任务​，比如[理解报销项目] -> [定位目标表格] -> [确认表格结构] -> [格式化并插入数据]​。同时，模型会判断当前需要解决哪个子任务，以及需要调用哪个工具来获取信息，比如决定首先调用“数据库查询工具”来列出表格现有行，以了解数据结构。

  • 行动：调用工具

根据思考阶段的决策，智能体会​​自主生成指令，并调用相应的外部工具或API。这正是智能体超越纯文本生成，能够“采取行动”的关键。在图1中，这个行动体现为AI向计算机发出“List rows in spreadsheet ‘TravelExpenses’”的指令。

  • 观察：获取工具结果

行动之后，智能体会​接收工具执行后的反馈结果​。这个结果可能是结构化的数据（如JSON格式的表格行），但为了便于模型理解，通常会将其转换为​​自然语言描述。这个结果会被添加到对话上下文中，作为下一轮“思考”的新输入。图1中，计算机返回的“Rows”就是一次观察结果。

  • 循环与判断

“观察”到的结果会促使循环进入下一轮。智能体会再次进行“思考”，评估现有信息是否足以完成任务。如果不能，它将规划下一步行动（例如，在了解表格结构后，下一步是“添加行”）；如果能，则​​合成所有步骤得到的信息，生成最终结论并结束循环。循环的终止条件通常是任务完成、达到最大迭代次数或遇到无法解决的错误。

下图（图2）清晰地勾勒出了这个包含人类、AI、计算机三方的核心循环流程。

工程实现的关键：从“灵活响应”到“可解析指令”

理想很美好，但工程实现面临一个核心挑战：大语言模型的输出是灵活的自然语言，而计算机系统需要的是结构化、可精准执行的指令。如果每次都需要人类手动解读AI的响应并执行操作（例如，手动复制AI生成的代码去运行），就完全失去了自动化的意义。

我们的目标是：人类用自然语言为Agent描述一个目标，大模型自动将目标拆解为一个个“调用（Call）”，这些调用能自动转换为对计算机系统的操作指令，并执行，直到完成目标。

具体来讲，我们给大模型一个输入提示（Prompt），得到大模型的响应（Response）。接下来，​我们需要能自动地将这个响应解析（Parse）成一个具体的、可程序化执行的行动（Action） 。行动执行后，将其结果作为下一轮提示的一部分，继续循环。

其中的关键在于如何控制大模型的输出，使其便于被后续程序解析。 解析过程依赖的是传统的、确定性的软件工程代码，它无法可靠地处理千变万化的自然语言表达。为了解决这个问题，我们通常在给大模型的“系统提示（System Prompt）”中做出严格约定，要求其以特定的结构化格式（例如JSON、XML或带有明确标记的文本）进行响应。这样，一个独立的“解析器（Parser）”模块就能可靠地从响应中提取出“工具名称”和“调用参数”，然后交给“执行器（Executor）”去调用对应的API。

下图（图3）则从工程实现视角，进一步拆解了“Agent Loop”内部的一个完整迭代步骤，特别是包含了“解析响应（Parse Response）”这个关键环节。

此图详细说明了智能体在一次循环内的微观步骤：1. 构建提示；2. 生成响应；3. ​解析响应；4. 执行动作；5. 将结果转换为字符串；6. 判断是否继续循环。它明确了“解析”是将大模型的语言响应转化为机器可执行动作的必经桥梁，是实现全自动闭环的核心。

通过将灵活的自然语言推理与确定性的程序化执行相结合，智能体循环为AI赋予了一只“无形之手”，使其能够主动探索环境、操作工具，并最终独立完成复杂的、多步骤的数字化任务。这，便是智能体强大能力的自动化基石。
