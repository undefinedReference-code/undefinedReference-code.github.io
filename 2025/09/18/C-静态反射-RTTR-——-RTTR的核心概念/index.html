C++静态反射 RTTR —— RTTR的核心概念

概要

好久没更新这个系列了，第三个部分我们介绍RTTR的核心概念，这里主要介绍六个概念，rttr::type、rttr::variant、rttr::property、rttr::method、rttr::constructor、rttr::registration，简单来讲，这六个概念的作用如下所示：

  • rttr::registration：注册机制，把类型/成员信息写入反射数据库
  • rttr::type：类型信息容器，类似 C# 的 System.Type​
  • rttr::variant：万能值容器，类似 C# 的 object 或 dynamic​
  • rttr::property：属性访问器，类似 C# 的 PropertyInfo/FieldInfo​
  • rttr::method：方法调用器，类似 C# 的 MethodInfo​
  • rttr::constructor：构造函数调用器，类似 C# 的 ConstructorInfo​

property

在 C++ 的 RTTR 和 C# 的反射系统中，property（属性）对象的作用是为程序员提供一种在运行时动态访问和操作类成员变量或属性的方法。通过 property，你可以在不知道具体类型细节的情况下，实现对对象成员的读取和修改。

method

method（方法）在反射系统中（无论是 RTTR 还是 C#）的作用是为程序员提供一种在运行时动态调用类成员函数或静态函数的能力。通过 method 对象，开发者可以在不知道具体类型或函数签名的情况下，通过统一的接口获取方法信息、查看参数类型和返回类型，并传入参数来执行目标函数。这使得诸如插件系统、脚本绑定、序列化、对象编辑器等需要运行时灵活扩展和调用的场景变得可能。method 是实现类型无关、动态方法调用的基础桥梁。
