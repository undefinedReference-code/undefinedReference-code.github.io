一文真正看懂std::move和std::forward

左值、右值、左值引用和右值引用

在最开始，我们介绍4个概念，这四个概念会用于我们后续理解move和forward的原理。

定义

1. 左值（Lvalue）

左值是指一个表达式结束后仍然存在的对象或变量。左值可以出现在赋值运算符的左边或右边。

特点：

  • 有名字的变量或对象。
  • 可以取地址。

2. 右值（Rvalue）

右值是指一个表达式结束后不再存在的临时值。右值只能出现在赋值运算符的右边。

特点：

  • 通常是临时对象或字面量。
  • 不能取地址。

3. 左值引用（Lvalue Reference）

左值引用是对左值的引用，使用 &​ 符号声明。左值引用必须绑定到一个左值。

特点：

  • 只能绑定到左值。

4. 右值引用（Rvalue Reference）

右值引用是对右值的引用，使用 &&​ 符号声明。右值引用必须绑定到一个右值。

特点：

  • 只能绑定到右值。

联系

左值和右值是值分类（Value Categories）、左值引用和右值引用是类型（Types）。 Categories和Types反映的是一个变量的不同方面。下面我们会看到，可能存在左值的右值引用，也存在右值的右值引用。

举个例子看看

首先看第一个调用func(i)。这里i是一个左值，类型是int。当传递i给func的时候，因为i是左值，所以只能匹配到接受左值引用的版本，也就是void func(int &t)，所以输出“int &”。

第二个调用是func(j)。j是i的左值引用，也就是j本身也是一个左值。所以同样的情况，这里调用的是func(int&)，输出“int &”。

第三个调用是func(1)。这里的1是一个右值，所以会匹配到接受右值引用的版本func(int&&)，输出“int &&”。

第四个调用是func(k)。这里可能容易出错。虽然k被声明为右值引用int&&，但k本身作为变量，它是一个左值。因为右值引用变量在表达式里是左值，它的类型是右值引用，但它本身作为具名的变量，具有持久性，所以会被当作左值处理。所以当传递k给func的时候，会调用接受左值引用的版本func(int&)，所以输出“int &”。

从上面可以看到，右值引用可以是一个左值。

左值返回值：

    • 函数返回一个左值引用，指向某个已经存在的变量。调用这个函数后，可以通过返回值对原变量进行修改。
    • 左值引用返回值是左值。

  2. 返回左值 (T​):

    • 例如：int func2() { return g_i; }​
    • 函数返回一个左值（非引用），调用这个函数后得到一个值的副本。
    • 左值返回值是右值。

右值返回值：

  1. 返回右值引用 (T&&​):

    • 例如：int&& func4() { return std::move(g_i); }​
    • 函数返回一个右值引用，通常用于转移语义，表示返回的对象可以被移动而不是拷贝。
    • 右值引用返回值是右值。

  2. 返回临时值:

    • 例如：int func2() { return 1; }​
    • 函数返回一个临时值（右值），如字面值或临时创建的对象。
    • 临时值是右值。

万能引用

在C++11及以后的版本中，万能引用（Universal References）是一种特殊的引用类型，它可以根据传递的参数类型自动推导为左值引用或右值引用。

实例：

  • 万能引用的格式为：T&&​，其中T​是一个模板参数。
  • 万能引用只能出现在模板函数或模板类中。

效果：

  • 左值传递：如果传递给模板函数的参数是左值，T​被推导为左值引用类型T&​，因此T&&​会折叠为T& &​，根据引用折叠规则，最终会变成T&​。因此，t​是左值引用。
  • 右值传递：如果传递给模板函数的参数是右值，T​被推导为非引用类型，T&&​保持为右值引用。因此，t​是右值引用。

需要注意的是，万能引用的格式必须是T&&，其中T是一个模板参数。下面这样的格式是一个右值引用，而不是一个万能引用

引用折叠

引用折叠规则在C++中定义了当多重引用（引用的引用）出现在代码中时，如何将它们折叠成单一的引用类型。具体规则如下：

  1. ​T& &​、T& &&​、T&& &​都折叠为T&​。
  2. ​T&& &&​折叠为T&&​。
